### Task 1: Keychain identity store (`internal/keystore`)

**Files:** Create `internal/keystore/keystore_darwin.go`, `keystore_other.go`,
`keystore_darwin_test.go`.

A darwin store for the age identity in the login keychain via the `security` CLI
(injected runner, like `internal/backend/keychain`). API:
```go
type Store struct{ run runner } // runner func(args ...string) ([]byte, error)
func New() *Store                // production: real `security`
func NewWithRunner(run runner) *Store
func (s *Store) Store(identity []byte) error  // security add-generic-password -U -s agentvault -a identity -w <bytes> -T /usr/bin/security
func (s *Store) Read() ([]byte, error)         // security find-generic-password -s agentvault -a identity -w ; ErrNotFound if absent
```
GOAL for the ACL: avd's silent `Read` must NOT pop a GUI prompt — use `-T /usr/bin/security`
(or `-A`); pick whichever the test/manual confirms is prompt-free and document it.
SECURITY: the identity passes via argv to `security` (brief same-UID `ps` window —
acceptable under the cooperative-agent threat model; document it). Errors carry no key
bytes. Non-darwin stub: `Store`/`Read` return a clear "requires macOS" error.

TDD: mock-runner tests — `Store` issues the right args; `Read` returns the value;
absent item → `backend.ErrNotFound` (reuse it or a local sentinel); round-trip.

Commit: `feat(keystore): login-keychain age-identity store (security CLI)`.

### Task 2: Tiered provisioning (`internal/provision`)

**Files:** Modify `internal/provision/provision.go` + test.

Extend `Options` with a tier choice and the keychain sink:
```go
type Tier string // "enclave","keychain","plaintext"
type Options struct {
	Dir            string
	Rotate         bool
	Tier           Tier                          // "" = auto
	RequireEnclave bool
	Wrap           func([]byte) ([]byte, error)  // enclave wrap (injected)
	KeychainStore  func([]byte) error            // keystore.Store (injected)
}
type Result struct{ Tier Tier; VaultPath, IdentityPath string; Created bool }
```
Selection:
- `auto` (default): try `Wrap` (enclave) → on success tier=enclave (write `identity.enc`);
  on ANY wrap error → tier=keychain (call `KeychainStore`) — NEVER auto-plaintext.
- `enclave` + `RequireEnclave`: wrap failure is a hard error (no downgrade).
- `keychain`: store the generated identity via `KeychainStore` (no on-disk identity file;
  `IdentityPath` = "keychain:agentvault/identity").
- `plaintext`: write `identity.txt` 0600.
All tiers still write the empty `vault.age` atomically (unique temp + rename, 0600) and
are idempotent (no clobber unless `Rotate`). `Result.Tier` reports what was chosen.
SECURITY: never log/return identity bytes; downgrade is the caller's to announce.

TDD: stub `Wrap` (success→enclave; error→fallback keychain via a recording
`KeychainStore`), `RequireEnclave` error path, explicit `keychain`/`plaintext`,
idempotency, vault decrypts for the plaintext case.

Commit: `feat(provision): tiered identity (enclave->keychain, plaintext opt-in)`.

### Task 3: Per-tier session unwrapper + avd wiring + discovery

**Files:** Modify `cmd/avd/main.go` (+ small helpers); extend `internal/client/e2e_test.go`.

The session unwrapper (Task 3/4 of the previous feature) is set for EVERY local-vault
tier; for non-enclave tiers it wraps the Touch ID itself:
- enclave: `unwrapper = func() ([]byte,error){ blob:=ReadFile(identity.enc); return unwrap(blob) }` (self-prompting via the key ACL; `mapEnclaveErr`).
- keychain: `unwrapper = func() ([]byte,error){ if err:=presence.Prompt("Unlock AgentVault"); err!=nil {return nil,err}; return keystore.Read() }`.
- plaintext: `unwrapper = func() ([]byte,error){ if err:=presence.Prompt(...); err!=nil {return nil,err}; return ReadFile(identity.txt) }`.
So `unlockWithUnwrapper` works uniformly; presence is injected into the non-enclave
closures. With NO local vault, no unwrapper (unlock = plain `presence.Prompt`, unchanged).

Discovery on startup (no `AV_AGE_*` env): detect the tier by what exists under the
config dir — `identity.enc`→enclave, else keychain item present→keychain, else
`identity.txt`→plaintext, else none. Wire the matching unwrapper + register
`agefile.New(session, vaultPath)`.

`makeProvisioner` (the live `setup`): pass `Wrap=wrap`, `KeychainStore=keystore.Store`;
after a `Created` provision, wire the unwrapper for the RESULT's tier and register the
backend live. Honor `av setup` flags (Tier/RequireEnclave) from SetupParams (Task 5
extends the proto).

Track the ACTIVE TIER + enclave-availability on the Server (for `version`): record which
tier the unwrapper represents.

TDD: extend e2e — with `AV_TEST_ENCLAVE=stub` (enclave path) AND a separate run forcing
keychain (stub keystore) under a temp HOME: `setup`→`add`→`unlock`(stub)→`run` masks the
value; assert the active tier.

Commit: `feat(avd): per-tier session unwrapper + tier discovery + live setup`.
