## Problem

After `brew upgrade agentvault`, the running `avd` keeps serving the OLD code until
manually restarted (a running process keeps its exec'd inode; `brew upgrade` doesn't
restart services). Worse, `av` can have autostarted a *detached* `avd` outside launchd
that `brew services restart` can't touch. Result: a stale daemon (the bug we hit).
`av version` now *detects* this, but the user still has to fix it by hand.

## Goal

`av` heals the skew automatically: when the daemon's version differs from `av`'s, `av`
shuts the old daemon down and lets the new binary take over, then retries тАФ so a plain
`brew upgrade` "just works" with no manual `brew services restart`.

## Design (decisions)

- **Skew detection:** the client, on its dial-for-work path, fetches avd's version via
  the existing `version` RPC and compares to `av`'s own `version`.
- **Heal trigger guard:** heal ONLY when `av.version != avd.version` AND neither is the
  sentinel `"dev"`. A dev build (unstamped) must never auto-restart a release daemon
  (and vice-versa) тАФ that would loop forever in a mixed dev/brew setup. Release-vs-
  release skew is exactly the `brew upgrade` case.
- **Heal action (interactive, i.e. `!NoPrompt`):** call a new `shutdown` RPC тЖТ the old
  daemon exits gracefully тЖТ re-dial via the EXISTING autostart/retry path (launchd
  respawns the new binary, or `av` autostarts it) тЖТ confirm the version now matches тЖТ
  retry the original operation. Print one line: `agentvault: daemon upgraded тАФ restarting`.
- **Bounded:** heal at most ONCE per `av` invocation. If still skewed after the heal,
  proceed with a LOUD warning (never loop, never hang).
- **Agent path (`AV_NO_PROMPT`):** do NOT auto-restart (an agent must not cause the
  surprising side effect of killing the user's daemon mid-session). Instead return a
  clear error тЖТ `av` prints `avd outdated (vX) vs av (vY) тАФ ask a human: brew services
  restart agentvault` and exits non-zero (agent pauses). Consistent with the pause
  philosophy.
- **Keep** the detached autostart (one-off convenience); the heal covers the stray case
  (shutdown the stray тЖТ next dial autostarts the new binary).
- **`shutdown` RPC** is peer-cred gated like every RPC (same-UID only тАФ no new exposure;
  a same-UID DoS via `shutdown` is no worse than `pkill avd`, which is out of scope).

## Security

No secret/identity involved. `shutdown` only stops the process. Restart relocks the
session (issued values + identity zeroized) тАФ strictly safe. Peer-cred unchanged.

### Task 2: client version-skew auto-heal
**Files:** `internal/client/client.go` (+ a heal helper), `cmd/av/main.go` (thread the
heal into run/read/add/rm/unlock/lock/status via `dialClient`/the work path), tests.
- Add `var version` is in `cmd/av` already (Task 5 prior). Pass `av`'s version into the
  client (e.g. `client.New(path).WithVersion(version)`), plus the existing `WithNoPrompt`.
- A client method `ensureFresh() error` (called once before the real op on a connection):
  `v, err := c.Version()`; if unreachable, ignore (autostart will bring up the right one);
  if `skew(c.version, v.Version)` (both non-"dev", differ):
  - if `c.noPrompt`: return a typed `ErrDaemonOutdated{Av, Avd string}`.
  - else: print the restart note to stderr; `c.Shutdown()` (ignore its connection-drop
    error); wait (bounded, e.g. тЙд5s polling) for a daemon whose `Version()` == c.version
    (re-dial uses the existing autostart/retry); if it comes up matched тЖТ return nil
    (caller proceeds); if not тЖТ print a LOUD warning and return nil anyway (don't block).
  Heal at most once (a `healed` bool guard).
- Wire `ensureFresh()` into the run/read/add/rm/unlock/lock/status client entry points
  (one call at the start; `av version` itself does NOT auto-heal тАФ it just reports).
- `cmd/av`: map `ErrDaemonOutdated` to a clear stderr message + non-zero exit (reuse
  exitGeneric or a new code; a clear message is the point).
- TDD: client/daemon integration test тАФ start a real test daemon reporting an OLD
  version (inject via SetVersion), client with a NEWER version: interactive heal calls
  Shutdown (assert the daemon teardown fired) ; NoPrompt returns `ErrDaemonOutdated`.
  Also: matched versions тЖТ no heal, no shutdown. dev-on-either-side тЖТ no heal.
- Commit: `feat(av): auto-heal daemon on version skew (agents surface, humans restart)`.
